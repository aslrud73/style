# 별쫑 화풍 라이브러리 — 배포 안내

## 📦 구성 파일
- `index.html` — 앱 본체 (PIN + 화풍 그리드 + 추가/수정/삭제 + Firebase 동기화)
- `manifest.json` — PWA 설정 (홈화면 추가용)
- `sw.js` — 서비스 워커 (오프라인 셸 캐싱)
- `icon-192.png`, `icon-512.png` — 앱 아이콘

---

## 🔑 1. PIN 번호 바꾸기 (선택)
`index.html` 안에서 아래 줄을 찾아 원하는 4자리로 바꾸세요.
```js
const APP_PIN = "1234"; // ← 여기
```
바꾼 뒤 저장 → 다시 배포(GitHub 푸시)하면 적용됩니다.

---

## 🚀 2. Vercel 배포
기존 방식 그대로:
1. 이 폴더(5개 파일)를 GitHub 저장소에 올림
2. Vercel에서 해당 저장소 연결 → 자동 배포
3. 받은 URL 접속 → PIN 입력 → 사용

> PWA(홈화면 추가)는 **https에서만** 작동합니다. Vercel은 https라 자동으로 됩니다.

---

## 🔒 3. Firestore 보안 규칙 (중요 · 30일 안에 꼭)
지금은 "테스트 모드"라 **30일 후(2026-07-19) 자동으로 잠겨** 데이터를 못 읽고 못 쓰게 됩니다.
그 전에 규칙을 바꿔야 계속 쓸 수 있습니다.

### 방법
Firebase 콘솔 → Firestore Database → **규칙(Rules)** 탭 → 아래로 교체 → **게시**

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /styles/{doc} {
      allow read, write: if true;
    }
  }
}
```

### ⚠️ 솔직한 한계
- 위 규칙은 "누구나 읽기·쓰기 허용"입니다. PIN은 앱 화면 진입만 막을 뿐, **DB 자체는 URL+키를 아는 사람이면 접근 가능**합니다.
- 혼자 쓰고 URL을 공유하지 않으면 현실적으로 충분합니다.
- 더 강한 보안이 필요해지면 그때 Firebase 인증(익명 로그인 등)으로 올리면 됩니다. 지금은 과설계라 보류.

## 🖼️ 4. 화풍 등록 방법 (완성 후 사용법)
1. S3에 이미지 업로드 → 이미지 URL 복사
2. 앱에서 **"화풍 추가"** 클릭
3. 폼 작성: 이름 / 카테고리 / 이미지 URL(붙여넣기) / 태그 / 프롬프트
4. **저장** → 모든 기기(PC·iPad·폰)에 자동 동기화
5. 카드 클릭 → 프롬프트 복사 / 수정 / 삭제

---

## 📱 5. 아이폰/아이패드 홈화면에 앱처럼 설치
1. Safari로 Vercel URL 접속
2. 공유 버튼 → "홈 화면에 추가"
3. 아이콘이 생기고, 열면 앱처럼 전체화면으로 뜸

(아이콘은 임시로 만든 것이라, 원하면 나중에 직접 만든 이미지로 `icon-192.png`/`icon-512.png` 교체 가능)
